## 一、你的 QA 经验，在 AI 测试里直接能用

先看这张表——你不是从零开始，而是带着底子来的：

| **你已有的 QA 能力**                                   | **在 AI 测试中怎么用**                 |
| ------------------------------------------------------------ | -------------------------------------------- |
| ------------------------------------------------------------ |                                              |
| 测试用例设计                                                 | → 评测数据集（Gold Set）设计                |
| -                                                            | -                                            |
| 边界值分析                                                   | → 对抗性测试（Prompt 注入、越狱、边界输入） |
| 回归测试                                                     | → Prompt / 模型变更后的回归评测             |
| 自动化测试                                                   | → 评测流水线（DeepEval / RAGAS）            |
| 缺陷追踪                                                     | → Bad Case 分析与闭环                       |
| CI/CD 集成                                                   | → 评测流水线集成到 CI/CD                    |
| 性能测试                                                     | → 延迟 / 吞吐 / 成本基准测试                |
| 测试报告                                                     | → 评测报告（准确率 / 幻觉率 / 一致性）      |

 **一句话** ：AI 测试不是全新领域，是你已有方法论在 AI 场景下的升级版。

## 二、为什么现在值得转

* **传统 QA 在收缩** ：手工测试被 AI 取代，大厂推"开发自测 + AI 辅助"，HC 持续缩减。
* **AI 质量岗是蓝海** ：大多数 AI 团队找不到"既懂测试方法论又懂 AI 的人"。据公开招聘平台样本观察，同级别 AI 测试/评测岗薪资普遍高于传统 QA。
* **策略** ：先以"AI 质量工程"拿到第一份 AI 岗位 offer，入职后再往评测平台、安全、后端方向拓宽。评测是入口，不是天花板。

## 三、这门课怎么学

 **核心设计** ：讲师负责准备一个可运行的 AI 应用（**企业知识库 Agent**）及配套测试环境，你 100% 精力搭质量体系——不用写业务代码。

 **标准 5 周（4 周强化 + 1 周求职冲刺）** ，基础薄弱可额外加 1 周预科：

| **周次**                                                         | **做什么**                 | **你拿到什么**            |
| ---------------------------------------------------------------------- | -------------------------------- | ------------------------------- |
| ---------------------------------------------------------------------- |                                  |                                 |
| Week 1                                                                 | AI 测试基础 + Prompt/API 回归    | 失败模式地图 + 回归测试套件     |
| -                                                                      | -                                | -                               |
| Week 2                                                                 | RAG 质量保障                     | RAG 评测套件 + 评测报告         |
| Week 3                                                                 | Agent E2E + 安全对抗测试         | E2E 自动化套件 + 安全测试包     |
| Week 4                                                                 | 评测平台化 + 可观测性 + 发布门禁 | 质量平台 + 评测看板             |
| Week 5                                                                 | 求职冲刺                         | AI 转岗简历 + 项目讲稿 + 作品集 |

> **预科（Week 0，选修）** ：面向有接口测试经验但缺 Python / Docker 基础的同学，补 Python 基础、Pytest 入门、Docker 操作和 LLM 最小概念。预科不是零基础编程课。

每周交付物都能单独写进简历。5 周结束后你手上有一套完整的 AI 质量工程项目。

 **不教什么** ：不教前端开发（React/Vue）、不从零搭 RAG/Agent 系统、不讲模型训练/微调、不涉及 GPU 集群。课程聚焦质量工程，不是全栈开发课。

## 四、谁适合 / 谁不适合

 **适合** ：有接口测试经验（Postman / HTTP 能看懂），或有基础脚本经验（Python / Shell / Java 任一），能用命令行。

 **不适合** ：纯手工功能测试、完全零脚本基础、不愿动手写代码（每周都有交付物）。

> 如果你有接口测试基础但缺 Python / Docker 经验，可以参加 Week 0 预科。如果完全没有脚本基础，建议先自学 1-2 周 Python 再考虑入班。
