## 整体项目规划：企业级AI电商供应链知识库

### 一、项目定位与架构原则

项目有双重目标——既是一个**真实可用的企业级产品**，又是一个**QA 学习 AI 测试的教学平台**。架构设计必须同时满足：

1. **可观测性优先** — 每个模块都要暴露足够的测试接口和监控指标，QA 才有东西可测
2. **分层解耦** — 让 QA 能独立对某一层做评测，不需要理解全栈
3. **渐进式复杂度** — 系统从简单到复杂逐步构建，对应 Week 1 → Week 4 的学习节奏
4. **技术栈对齐** — 前端 Vue 3、后端 Python (FastAPI)，匹配学员现有能力，降低上手门槛

### 三、关键架构决策

| 决策点 | 选型 | 理由 |
|-------|------|------|
| 前端框架 | **Vue 3 + Vite + TypeScript** | 学员已有 Vue 经验，零上手成本 |
| UI 组件库 | **Element Plus** | Vue 生态最成熟的企业级组件库，表单/表格/对话框开箱即用 |
| 状态管理 | **Pinia** | Vue 3 官方推荐，轻量直觉 |
| 图表可视化 | **ECharts** | 评测看板需要丰富图表，ECharts 中文生态强 |
| 流式输出 | **SSE (Server-Sent Events)** | 比 WebSocket 轻量，适合 LLM 流式回答场景 |
| 后端框架 | **FastAPI (Python)** | 学员已有 Python 基础，异步性能好，自动生成 OpenAPI 文档 |
| RAG 框架 | **LangChain** | 社区大、文档多、面试出现频率最高 |
| 向量数据库 | **ChromaDB**（开发）→ **Milvus**（进阶） | ChromaDB 零配置启动，Milvus 企业级面试加分 |
| LLM 网关 | **自建 Python 薄封装层** | 面试能讲"多模型切换策略"，学员用 Python 能读懂源码 |
| 评测框架 | **Pytest + DeepEval + RAGAS** | 行业标准，学员 Python 可直接上手 |
| 前端测试 | **Vitest + Cypress** | Vitest 与 Vite 原生集成，Cypress 做 E2E |
| 部署 | **Docker Compose** | 学员本地能跑，演示方便 |
| CI/CD | **GitHub Actions** | 免费、学员都有 GitHub 账号 |

### 四、知识库业务域：电商供应链

选择电商供应链作为知识域，场景丰富且面试官容易理解：

| 知识模块 | 内容示例 | 测试价值 |
|---------|---------|---------|
| 采购管理 | 供应商准入标准、采购流程、合同模板 | 事实性问答、流程类测试 |
| 仓储物流 | 入库/出库 SOP、库存预警规则 | 数值计算、规则推理测试 |
| 订单履约 | 发货时效、退换货政策、异常处理 | 多轮对话、条件判断测试 |
| 品控质检 | 质检标准、不合格品处理流程 | 专业术语、表格/图片理解 |
| 供应商管理 | 评分体系、绩效考核指标 | 结构化数据提取测试 |

### 五、项目分阶段规划（与 QA 学习节奏对齐）

#### Phase 0：基座搭建（讲师独立完成，开课前）

讲师交付一个**最小可运行版本**：

- **后端**：FastAPI + 基础 RAG Pipeline（LangChain）+ LLM Gateway 封装（Claude + DeepSeek）
- **前端**：Vue 3 + Element Plus，包含问答对话页面 + 简易知识管理页面
- **数据**：10-20 篇电商供应链种子文档
- **基建**：Docker Compose 一键启动、OpenAPI 文档自动生成
- **关键**：故意保留可测试的问题——偶尔幻觉、检索不精准、Prompt 注入漏洞——给 QA 有东西可发现

#### Phase 1 (Week 1)：API 层测试 + Prompt 回归

**QA 学员交付：**
- API 自动化测试套件（Pytest + httpx/requests）
- Prompt 变更回归测试框架
- 失败模式分类地图（幻觉、拒答、跑题、格式错误等）
- 基础 Bad Case 收集流程

**讲师同步迭代：**
- 后端增加请求日志中间件、基础 tracing
- 前端增加回答质量反馈组件（👍👎），Vue 学员可参与此组件开发

#### Phase 2 (Week 2)：RAG 质量评测

**QA 学员交付：**
- RAG 评测数据集（Gold Set：50-100 条问答对）
- 使用 RAGAS / DeepEval 的自动化评测套件（Python）
- 评测指标看板前端页面（Vue + ECharts，展示 Faithfulness / Answer Relevancy / Context Precision）
- 分块策略对比实验报告

**讲师同步迭代：**
- 添加 Rerank 模块
- 支持多种分块策略切换（给 QA 做 A/B 测试素材）
- 后端增加 `/eval` API 端点

#### Phase 3 (Week 3)：E2E + 安全对抗

**QA 学员交付：**
- 前端 E2E 自动化测试（Cypress，覆盖多轮对话流程）
- Prompt 注入/越狱攻防测试套件（Python）
- 权限边界测试（不同角色看到不同知识）
- Agent 工具调用链路测试（如果引入了 Agent）

**讲师同步迭代：**
- 引入简单 Agent 能力（如自动查库存、查订单状态）
- 增加 RBAC 权限控制
- 前端增加角色切换功能

#### Phase 4 (Week 4)：平台化 + CI/CD 集成

**QA 学员交付：**
- 评测流水线集成到 GitHub Actions CI
- 质量门禁（评测不过则阻断部署）
- 可观测性看板（Vue + ECharts 质量 Dashboard）
- 性能基准测试（延迟、吞吐、成本）

**讲师同步迭代：**
- 完善前端管理后台（知识文档 CRUD、版本管理）
- 增加系统配置页面（模型切换、分块策略选择）

#### Phase 5 (Week 5)：求职冲刺

- 项目 README 和架构文档完善
- 个人简历中的项目描述打磨
- 模拟面试 + 项目讲解演练
- GitHub 作品集整理

### 九、面试亮点设计（刻意埋入）

让学员在面试中能**讲出深度**的点：

1. **多模型 A/B 评测** — "我们对比了 Claude 和 DeepSeek 在供应链场景下的准确率，Claude 在复杂推理上高 15%，DeepSeek 在成本上低 80%"
2. **RAG 分块策略实验** — "我设计了 3 种分块策略的对比实验，发现语义分块比固定 512 token 分块的检索准确率高 23%"
3. **Prompt 注入防御** — "我构建了 50 个对抗性测试用例，发现系统存在 3 类注入漏洞并推动修复"
4. **质量门禁** — "我在 CI 中集成了评测流水线，Faithfulness 低于 0.85 就阻断部署"
5. **Bad Case 闭环** — "建立了 Bad Case → 根因分析 → 修复 → 回归验证的完整闭环"
6. **Vue 评测看板** — "我用 Vue 3 + ECharts 开发了评测数据可视化看板，实时展示各维度质量指标"
7. **E2E 自动化** — "用 Cypress 覆盖了多轮对话、流式输出、权限控制等核心场景的端到端测试"
